Functional programming & two track model - the basics for the "Railway Oriented Programming" 

						
																					 Use Case
												 ______________________________________________
												/                                              \
											 /                                                \
											|    operation 1    operation 2       operion 3    |
											|    _______         _________        ________     |
											|   /       \       /          \     /         \   |
						Input --> |  | Import  | --> | CSV parse | -> | Update db |  | --> Output
											|   \_______/       \_________/      \_________/   |
											|                                                  |
											|                                                  |
											 \                                                /
												\______________________________________________/

Use case is a function is build from smaller functions combined into a pipeline, each representing one step in a data flow.

One function's output in the input of another one

						
							def use_case
								update_db(csv_parse(import))
							end

We have to release whole process on error

Deviation the success track means the use case is failed

The result of operation decides about changing the track

						
							def step1
								puts 'executing #step_1'

Object-oriented implementation:

visit ChainIt gem: https://github.com/tier-tools/chainit 
						
							Result = Struct.new(:success, :value) do
								def success?
									success
								end
							end

							class ChainIt
								def chain
									return self if @skip
									@result = yield @result&.value
									@skip = true unless @result.success?
									self
								end

								def result
									@result
								end
							end
						
					
						
							def step1
								puts 'executing #step_1'
